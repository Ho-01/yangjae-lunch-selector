# ADR 0005: 음식 카테고리와 날씨 프로필 분리

- 상태: 승인
- 날짜: 2026-07-24

## 배경

기존 `t_menu_type`은 화면의 메뉴 종류와 날씨별 확률 설정을 동시에 표현했다.
Google Places의 음식 분류를 같은 필드에 넣으면 `한식` 같은 음식 계열이 날씨 확률의
입력이 되는 의미 충돌이 발생한다.

## 결정

- `food_category`는 검색·필터·표시에만 사용하고 확률 계산에 사용하지 않는다.
- `weather_profile`은 날씨별 확률 계산에만 사용한다.
- Google의 `primaryType`과 `types`는 `food_category`로만 변환한다.
- 내 주변 식당은 신뢰할 수 있는 별도 날씨 성향이 생기기 전까지 중립 프로필을 사용한다.
- 기존 `t_menu_type` 데이터는 새 `t_weather_profile`로 같은 ID를 유지해 복사한다.
- 배포 호환 기간에는 기존 테이블과 컬럼을 유지하고 동기화 트리거를 사용한다.

## 결과

양재역 주변 메뉴의 기존 가중치는 유지된다. 내 주변 카테고리 선택은 확률을 바꾸지
않는다. 향후 Spring 서버에서도 `foodCategory`와 `weatherProfile`을 독립 계약으로
제공한다. 기존 `t_menu_type` 제거는 모든 소비자가 새 계약으로 전환된 뒤 별도
마이그레이션에서 수행한다.
