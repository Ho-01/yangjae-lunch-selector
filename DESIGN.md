# 식사가챠 디자인 규칙

이 문서는 shadcn/ui를 사용하면서 식사가챠다운 일관성을 유지하기 위한 기준이다.
컴포넌트 구현 방법은 shadcn/ui를 따르고, 제품의 색상·밀도·문장은 이 문서를 따른다.

## 시각 방향

- 기본 바탕은 편안하고 따뜻한 `Stone`, 핵심 행동과 당첨 순간은 식욕을 돋우는
  `Orange`로 표현한다.
- 장식보다 `모드 선택 → 후보 확인 → 돌리기 → 결과 이해` 흐름을 우선한다.
- 한 화면에서 강한 주황색 강조는 원칙적으로 하나만 둔다.

## 색상

실제 색상값은 `lunch-wheel/src/index.css`의 shadcn/ui 의미 토큰에서만 관리한다.

- `background`, `card`, `secondary`, `muted`: 화면 구조와 보조 정보
- `primary`: 돌리기, 저장, 참여처럼 다음 단계로 진행하는 대표 행동
- `accent`: 선택됨, 가벼운 강조, 호버 상태
- `destructive`: 삭제와 되돌리기 어려운 행동
- `foreground`, `muted-foreground`: 본문과 보조 문장
- `border`, `input`, `ring`: 경계, 입력 상태, 키보드 포커스
- `chart-1`~`chart-5`: 룰렛, 색종이처럼 여러 단계의 Orange가 필요한 시각화

컴포넌트에서 새 HEX, RGB, OKLCH 값을 직접 추가하지 않는다. 새로운 의미가 필요하면
먼저 기존 토큰으로 표현할 수 있는지 확인하고, 반복되는 경우에만 이 문서와
`index.css`에 의미 토큰을 함께 추가한다. 사용자 지정 메뉴 색상과 Canvas로 생성하는
공유 이미지는 예외지만 이 문서의 팔레트와 명도 대비를 유지한다.

## 컴포넌트

- 버튼, 입력, 선택, 체크박스, 스위치, 대화상자는
  `lunch-wheel/src/components/ui/`의 shadcn/ui 컴포넌트를 사용한다.
- 대표 행동은 `default`, 보조 행동은 `secondary` 또는 `outline`, 낮은 우선순위는
  `ghost`, 삭제는 `destructive` 변형을 사용한다.
- 제품 고유 UI는 shadcn/ui 프리미티브를 조합한다. 같은 조합이 반복되면 공통
  컴포넌트로 올린다.
- 터치 영역은 최소 44×44px, 키보드 포커스 링은 항상 보이게 유지한다.

## 레이아웃과 문장

- 모바일 320px부터 가로 스크롤 없이 핵심 행동을 완료할 수 있어야 한다.
- 모바일 본문은 원칙적으로 13px 이상, 주요 설명은 14px 이상을 사용하고 1.5 이상의
  줄 간격을 유지한다. 상태 보조 문구처럼 제한적인 경우에만 11px을 허용한다.
- 모바일 버튼은 최소 44px 높이를 유지하고 긴 한글 문구가 잘리기보다 자연스럽게
  줄바꿈되도록 한다.
- 같이 고르기처럼 단계가 긴 화면은 부가 기능을 접을 수 있게 하고, 후보 선택과
  준비처럼 다음 결정에 필요한 조작을 먼저 읽을 수 있게 배치한다.
- 결과와 오류는 가장 중요한 내용부터 말하고, 사용자가 이어서 할 행동을 함께 제시한다.
- 400ms 이상 걸릴 수 있는 초기 작업은 진행 상태를 즉시 표시한다. 핵심 데이터와 부가
  데이터를 분리할 수 있으면 핵심 화면을 먼저 열고 나머지는 점진적으로 채운다.
- API, 캐시, 세션, 데이터베이스 같은 구현 용어를 사용자 화면에 노출하지 않는다.
- 아이콘은 Lucide SVG를 사용하고 장식용 이모지 문자를 직접 넣지 않는다.
- 음식 계열은 `음식` Orange 배지, 확률에 영향을 주는 날씨 특성은 `성향` Stone
  배지로 구분한다. 두 개념을 색상만으로 구분하지 않고 라벨과 아이콘을 함께 표시한다.

## 변경 확인

UI 변경 시 아래 항목을 확인한다.

1. 의미 토큰만 사용했는가
2. 대표 행동이 화면에서 하나로 명확한가
3. 320px와 390px에서 잘림과 가로 스크롤이 없는가
4. 키보드만으로 조작 가능하고 포커스가 보이는가
5. 로딩·빈 상태·오류·성공 상태가 사용자 문장으로 설명되는가

구현 후 `npm run lint`, `npm run build`, 사용자 흐름이 바뀌었다면
`npm run test:e2e`를 실행한다.
