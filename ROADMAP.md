# 제품 로드맵

이 문서는 아직 출시하지 않은 제품 방향과 우선순위만 관리한다.
이미 출시한 기능과 버전별 변경 사항은 [`CHANGELOG.md`](CHANGELOG.md)에서 확인한다.

마지막 검토일: 2026-07-24

## 관리 원칙

- `Now`는 현재 개발 중인 항목이다. 동시에 하나의 핵심 결과에 집중한다.
- `Next`는 다음 개발 후보로 합의됐지만 아직 착수하지 않은 항목이다.
- `Later`는 가치가 있지만 사용자 근거, 비용 또는 설계 검증이 더 필요한 항목이다.
- 착수하지 않은 항목에는 마일스톤 번호, 목표 버전, 확정 일정을 붙이지 않는다.
- 아이디어가 구체화되면 기대하는 사용자 결과와 완료 조건을 적고 `Now`로 옮긴다.
- 출시가 끝나면 이 문서에서 제거하고 `CHANGELOG.md`의 해당 버전에 기록한다.
- 구현 작업과 세부 요구 사항은 GitHub Issue 또는 PR에 연결한다. 로드맵을 작업 체크리스트로 사용하지 않는다.
- 우선순위는 주요 릴리스 전 또는 최소 분기마다 다시 검토한다.

## Now

### 내 주변 식당 탐색 개선

기대 결과: 사용자가 현재 위치 주변의 식당을 음식 종류와 실제 방문 가능성을 기준으로
빠르게 좁혀 룰렛 후보에 넣을 수 있다.

현재 범위:

- Google Places 식음료 타입을 앱의 한식·중식·일식·양식·아시아 음식·세계 음식·
  간편식·카페/디저트·주점·기타 체계로 변환
- 이미 불러온 결과에서 단일·복수 카테고리를 선택하는 모바일 흐름
- 분류되지 않았거나 Google 분류가 모호한 식당의 `기타` 처리
- 추가 Places 호출 없이 로컬 우선 필터링

완료 조건:

- Google 식음료 타입 스냅샷 전체가 누락 없이 앱 카테고리로 매핑된다.
- 알 수 없는 새 타입과 타입이 없는 식당이 오류 없이 `기타`로 표시된다.
- 모바일에서 카테고리 선택부터 후보 확인과 룰렛 실행까지 완료할 수 있다.
- 필터 결과가 없을 때 복구 방법이 표시되며 필터 변경이 API 호출을 만들지 않는다.

## Next

현재 확정된 다음 이니셔티브가 없다. `Later` 항목 중 사용자 가치와 위험 감소 효과가
가장 큰 항목을 검증한 뒤 선택한다.

## Later

### 점심방 의사결정 설명

기대 결과: 참여자가 좋아요와 거부권이 최종 후보에 어떤 영향을 주었는지 이해할 수 있다.

검토할 범위:

- 좋아요·거부권 반영 결과 설명
- 방 만료 시간 6시간이 실제 사용 패턴에 적합한지 검증
- 식이 제한과 알레르기를 불필요하게 노출하지 않는 공유 방식

### 점심방 안전 기능

기대 결과: 공개 범위가 넓어져도 사용자가 원치 않는 메시지와 참여자를 통제할 수 있다.

검토할 범위:

- 실제 사용량과 문제 사례를 바탕으로 신고·차단 필요성 판단
- 방 코드 공유 범위와 초대 방식 검토
- 로그인 없이 가능한 제재의 한계와 인증 도입 시점 검토

### 운영 준비

기대 결과: 사용량이 늘어도 장애와 데이터 위험을 파악하고 복구할 수 있다.

검토할 범위:

- Supabase Auth, 팀, 구성원 권한과 제한적인 RLS
- 애플리케이션 오류 모니터링과 운영 알림
- 백업·복구·데이터 보존·위치 개인정보 정책
- 성능·보안·접근성 릴리스 점검
- Spring 서버와 PostgreSQL로 이전할 때의 단계별 전환 검증

## 사용 근거 없이는 개발하지 않을 항목

- 소셜 피드, 공개 프로필, 배지와 과도한 게임화
- 최근 결과 조정 효과를 검증하기 전의 복잡한 개인화 설정
- 비용 통제를 약화하는 자동·고빈도 Places 새로고침
